# frontend-design

Origin: adapted from Everything Claude Code (ECC)

Use when:

- improving the editorial panel UI
- building new dashboard surfaces
- turning raw admin views into workflow-oriented interfaces

Core rules:

- commit to a visual direction instead of generic admin UI
- organize the page around operator tasks, not around database fields
- use hierarchy, spacing, typography, and contrast intentionally
- keep desktop and mobile usable

Project-specific guidance:

- the panel should support three things: preparing the next digest, reviewing old decisions, and capturing training signals
- “future” and “archive” workflows should feel distinct
- preview widgets should open into usable work surfaces, not dead summaries
